PanelProof

Check finished artwork against the source of truth — the proofing companion to Data Merge

v1.0.0
What this does. PanelProof opens your finished Illustrator files and checks them, field by field, against what they're supposed to say — the spec sheet, the price list, the approved data. It reads the live text out of .ai files and live-text PDFs and compares every value exactly. And it renders every file and looks at it, so it can also verify outlined or rasterized PDFs that have no readable text, and catch a substituted font or text running past its box. Where Data Merge builds the files, PanelProof proves them.
It never changes your files. PanelProof is read-only by design. It opens, checks, and closes without saving. If it finds something wrong, it reports it — fixing is Data Merge's job. A tool that both judges and edits is one you can't fully trust, so this one only judges.

What the results mean

passThe value matches the spec exactly. This is what you want.
presentThe value is in the file, but not where the template expected it (common with PDFs). Glance at the render to confirm it's in the right spot.
reviewCan't be read as text — usually an outlined or rasterized PDF. PanelProof renders it and reads the picture instead; you confirm against the render.
failThe spec value isn't in the artwork at all. Shown with expected-vs-found so the fix is obvious.

How you actually use it (just describe the job)

  • Proof these panels against the spec sheets before they go to the printer.
  • Did the labels come out right? Check them against this data.
  • QC this folder of artwork for me.
  • Verify these final PDFs say what they're supposed to.
  • Second set of eyes on these — do the numbers match?
1
Get the skill file from your team lead

You need panelproof.skill. Save it somewhere you can find it.

Note: If your file picker won't accept a .skill file, rename it to panelproof.skill.zip for the upload.
2
Turn on Code Execution in Claude

In Claude: Settings → Feature Preview → turn on "Code execution & file creation." Once only. (If you already use ColorProof or Data Merge, it's on.)

3
Upload the skill to Claude

Go to Settings → Capabilities → Skills → Upload skill and choose panelproof.skill. You'll then see PanelProof in your skills list.

Invoke it as /panelproof or just by describing the job ("proof these against the spec").
4
Deploy the engine to your Mac

Open Terminal and paste:

mkdir -p ~/.panelproof/dist
cp scripts/panelproof.jsx scripts/run.sh scripts/proofreport.js ~/.panelproof/
chmod +x ~/.panelproof/run.sh

Where are these scripts? Inside the .skill file, which is a zip. Rename panelproof.skill to panelproof.skill.zip, double-click to unzip, and run the commands from inside the unzipped panelproof folder.

Note: There's no backup step like other tools have — PanelProof never edits your files, so there's nothing to back up.
5
Make sure Adobe Illustrator is open

PanelProof drives the real Illustrator to read and render your files, so Illustrator must be running. The first run asks macOS for permission to control Illustrator — click OK (or re-enable later under System Settings → Privacy & Security → Automation).

6
Point Cowork at your folders

Share the artwork folder and wherever your source data lives (spec sheets, spreadsheet). Add folder → choose it → Allow Access.

7
Try it

With Illustrator open and a folder shared, say:

"Proof this folder against these spec sheets and show me the report."

Claude reads each file, renders it, scores every field, and opens a report with pass/fail and a proof image of each file. Anything it couldn't read as text, it will have looked at on the render and told you what it saw.

You're set. Hand over the artwork and the source of truth, and read the report it gives back.

Troubleshooting

Everything came back "review"
The files have no live text — they're outlined or rasterized PDFs. That's expected: PanelProof renders them and verifies off the picture. Read the renders in the report; that's the proof for those files.
A value shows "present" instead of "pass"
The value is in the file but not in the frame the template expected (common when a PDF reflows text). It's almost certainly fine — glance at the render to confirm it's sitting in the right place.
Something failed that looks correct to me
Check the expected-vs-found in the report. Usually it's a real but tiny difference (a missing space, a different unit, a stray character), or the spec value you gave it was off. PanelProof compares to the spec you provided, so confirm the spec first.
It "failed" a missing period (or similar) that's actually right
It shouldn't — PanelProof respects the source. If the spec has no period, artwork with no period passes. If you see this, the spec value handed in probably had the period; fix the spec, not the art.
No report appeared
Bring Illustrator to the front and dismiss any open dialog, then re-run. For a big folder, tell Claude to "run it backgrounded."
A font looks wrong on the render
That's the visual layer doing its job — the file's font isn't installed on your Mac, so Illustrator substituted it. Install the correct font and re-run. (This is exactly the kind of thing a text-only check would miss.)
Where are the proof images?
In a _proof_previews folder next to the artwork. It's scratch — delete it when you're done.
