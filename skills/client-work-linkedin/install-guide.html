Client Work → LinkedIn

Turn a finished client project into a ready-to-post LinkedIn post

v1.0.0
What this does. Point Claude at a project you just wrapped, a code repo, a folder of design files, or its brief in Notion or Drive, and it writes a LinkedIn post in a first-person agency-owner voice. It reads the context to understand what was built, who it was for, and what changed for the client, then tells that as a business story. No tech jargon, no press-release fluff, no em dashes, no invented numbers.

You pick a post type (problem-solution, before-after, client win, lesson learned, and so on) or let it draft two or three variants to choose from. This is a one-time setup. After it's added, you just talk to Claude.

How you actually use it (just describe the job)

  • Turn the project we just wrapped into a LinkedIn post.
  • Write a before-and-after post about this rebrand, don't name the client.
  • Give me three post options from this project so I can pick one.
  • Draft a client-win post from the brief in Notion.
  • Make a lesson-learned post out of what we figured out on this build.

Files
SKILL.md
1
Get the skill file

You need client-work-linkedin.skill. Download it from the catalog and save it somewhere easy to find (Downloads is fine).

Note: If your file picker won't accept a .skill file, rename it to client-work-linkedin.skill.zip before uploading, then rename it back if you need to.
2
Upload the skill to Claude

In Claude.ai, go to Settings → Claude Cowork → Upload skill, then pick your client-work-linkedin.skill file.

Once uploaded, you'll see Client Work → LinkedIn listed under your skills. That's it, the skill is now available in any Cowork session.

It's invoked as /client-work-linkedin or just by describing what you want ("turn this project into a LinkedIn post"). You don't need to type a version number.
3
Give Claude the project to write about

The skill writes from real context, so it needs to reach the project. Do whichever fits:

Design or packaging work: in Cowork, Add folder and point at the project folder (deliverables, brief, creative direction).

Code work: add the repo folder the same way so Claude can read the README and recent history.

Brief lives in Notion, Drive, or HubSpot: connect those under Settings → Connectors and the skill will pull the project context from there.

Nothing connected? You can also just paste the brief or a few sentences about the project into the chat. The skill works from whatever context it's given.
4
Ask for the post

Tell Claude what you want, for example:

"Turn the project in this folder into a before-and-after LinkedIn post. Don't name the client."

It will read the context, confirm the post type (or draft a couple of variants), and hand you a post ready to paste into LinkedIn. Edit a line or two and publish.

You're done. From here, just describe the job each time: "make a client-win post from this," "give me three options," "write a lesson-learned post." The skill handles voice, structure, and the banned-phrase cleanup.

Troubleshooting

It wrote about the wrong project, or guessed at details
It couldn't find solid context. Point it at the exact project folder, or paste the brief into the chat. The more specific the source, the sharper the post.
The post added emoji or hashtags I didn't want
The default is none of either. If they slipped in, say "no emoji, no hashtags" and it will redo it. Hashtags only appear when you ask for them.
It put a statistic in that we never measured
The skill is told not to invent numbers, but if one appears that isn't in your source, tell it to remove the figure and describe the change in words instead.
It named the client
By default it never names the client. If you want the name in, say so. To keep it anonymous, add "don't name the client" to your request.
The voice feels generic, not like us
If a Capsule brand voice guide is in the project or workspace, the skill will apply it. Point it at the guide, or paste a couple of posts that sound right and ask it to match that tone.
