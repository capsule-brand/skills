Illustrator Data Merge

One template, a list of records, one finished file per record

v1.0.0
What this does. You know Data Merge in InDesign — one layout, a spreadsheet, and it spits out a version per row. This brings the same idea to Adobe Illustrator. Give Claude one template .ai file and a list (a spreadsheet, spec sheets, or just typed values), and it builds one finished .ai per record, swapping the text, numbers, and even linked photos you choose — while leaving your fonts, colors, and layout exactly as designed. Back labels per flavor, shelf tags per SKU, name badges with headshots, certificates, multilingual versions, Nutrition Facts panels: same layout, many versions, done in one pass.

This is a one-time setup per Mac. After that, you just talk to Claude — no commands to memorize.

How you actually use it (just describe the job)

  • Build a back label for each of these wines — same layout, change the varietal, vintage, and ABV.
  • Here's a spreadsheet of 200 attendees — make a name badge for each with their headshot.
  • Generate a shelf tag per SKU from this price list.
  • Populate this Nutrition Facts panel for every flavor from these spec sheets.
  • Make a version of this poster for each city with its own date and venue.
1
Get the skill file from your team lead

You need a file called datamerge.skill. Ask whoever set this up for the latest version. Save it somewhere you can find it (Downloads is fine).

Note: If your file picker won't accept a .skill file, rename it to datamerge.skill.zip for the upload, then rename it back if needed.
2
Turn on Code Execution in Claude

Data Merge needs Claude's code feature on. In Claude: Settings → Feature Preview → turn on "Code execution & file creation." You only do this once.

3
Upload the skill to Claude

In Claude, go to Settings → Capabilities → Skills → Upload skill and pick your datamerge.skill file. Once uploaded, you'll see Illustrator Data Merge in your skills list, available in any Cowork session.

You can invoke it as /datamerge or just by describing the job ("make a label for each of these"). No version numbers to type.
4
Deploy the engine to your Mac

The skill includes a small engine that runs inside Illustrator. Copy it to a folder on your Mac once. Open Terminal and paste:

mkdir -p ~/.datamerge/profiles ~/.datamerge/dist
cp scripts/datamerge.jsx scripts/run.sh scripts/snapshot.sh scripts/report.js ~/.datamerge/
chmod +x ~/.datamerge/run.sh ~/.datamerge/snapshot.sh

Where are these scripts? Inside the .skill file, which is really a zip. Rename datamerge.skill to datamerge.skill.zip, double-click to unzip, and you'll find a scripts/ folder. Run the commands from inside the unzipped datamerge folder.

Note: ~/.datamerge/ is hidden in Finder on purpose — the engine lives there quietly and you never need to open it.
5
Make sure Adobe Illustrator is installed and open

Data Merge drives the real Illustrator app. There's no background mode — Illustrator must be running when you use it. Any recent CC version is fine.

The first time it runs, macOS asks if Claude can control Adobe Illustrator. Click OK. If you declined by accident, re-enable it under System Settings → Privacy & Security → Automation.

6
Point Cowork at your folders

In a Cowork session, give Claude access to the folders for the job: your template, the output folder, your data (spreadsheet or spec sheets), and any image folder (headshots, product shots). Add folder → choose it → Allow Access.

7
Try it

With Illustrator open and a folder shared, tell Claude:

"Show me the fields in this template, then build one test record so I can see it."

Claude opens a report in your browser showing the template's fields, and a proof image of the first result. If you see that, you're set.

You're done. From here just describe the job and hand over the template plus the list. Claude profiles the template once, builds a file per record, and shows you proofs before you ship.

Troubleshooting

No report appeared after a run
Two causes: (1) a dialog is blocking Illustrator — bring it to the front and dismiss any alert; (2) a big batch timed out — tell Claude to "run it backgrounded" and it uses a longer-timeout method.
Claude says it can't control Adobe Illustrator
Go to System Settings → Privacy & Security → Automation and allow the Claude/Cowork entry to control Adobe Illustrator. Toggle it off and on if it looks enabled but isn't working.
A field didn't change (Claude says "NOTFOUND")
The placeholder text the template profile expected wasn't in the file — usually the template was edited. Ask Claude to re-profile the template, or tell it the exact text that's there now.
A photo didn't swap (Claude says "NOIMG")
The image in the template must be a linked placed image with a name in the Layers panel (e.g. "headshot"). Select it, double-click its Layers row, give it a name, and make sure it's linked (not embedded). Then re-run.
Text runs past its box on one record
A long value overflowed a fixed-size frame. That's a template/layout decision, not a merge error — Claude flags it on the proof. Either shorten the text, or widen/auto-size that frame in the template and re-run.
How do I undo a batch?
Before any run that overwrites files, the whole output folder is backed up to ~/Desktop/Data Merge Backups/ with a timestamp. Copy that back over the originals. Nothing is permanently overwritten.
Where do my finished files go?
Into the output folder you chose, one .ai per record. A _previews subfolder holds the proof images — it's scratch, delete it before handing off the deliverables.
Fonts look wrong in a result
The fonts used in the template must be installed on your Mac, or Illustrator substitutes them. Install the template's fonts and re-run.
