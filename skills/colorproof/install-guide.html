ColorProof

Brand color preflight & auto-corrector for Adobe Illustrator

v2.1.0
What this does. ColorProof lets you ask Claude to check every color in a logo package against exact brand values — Pantone, CMYK, or hex — and fix anything that's off, across .ai, .pdf, .svg, .jpg, and .png files at once. Think of it as a spellchecker for brand color. You proof first, see a clear report of what's right and what's wrong, then approve the fix. Nothing changes without a backup taken first.

This is a one-time setup per Mac. After it's done, you just talk to Claude — no commands to memorize.
1
Get the skill file from your team lead

You need a file called colorproof.skill. Ask whoever set this up for the latest version — it should be named colorproof-v2.1.0.skill or similar. Save it somewhere you can find it (Downloads is fine).

Note: If your file picker doesn't accept .skill files, rename it to colorproof.skill.zip before uploading. Rename it back if needed after.
2
Enable Code Execution in Claude

ColorProof needs Claude's code execution feature turned on. In Claude.ai:

Settings → Feature Preview → turn on "Code execution & file creation"

You only need to do this once.

3
Upload the skill to Claude

In Claude.ai, go to Settings → Claude Cowork → Upload skill, then pick your colorproof.skill file.

Once uploaded, you'll see ColorProof listed under your skills. That's it — the skill is now available in any Cowork session.

The skill is invoked as /colorproof or just by describing what you want ("proof this package against our brand colors"). You don't need to type a version number.
4
Deploy the engine to your Mac

The skill includes an engine that runs inside Adobe Illustrator. You need to copy it to a folder on your Mac before ColorProof can use it. Open Terminal and run:

mkdir -p ~/.colorproof
cp scripts/colorproof.jsx scripts/run.sh scripts/report.js scripts/snapshot.sh scripts/acb_lab.py ~/.colorproof/
chmod +x ~/.colorproof/run.sh ~/.colorproof/snapshot.sh

Where are these scripts? They're inside the .skill file (which is a zip). If you haven't extracted them already, rename colorproof.skill to colorproof.skill.zip, double-click to unzip, and you'll find a scripts/ folder inside.

Note: The ~/.colorproof/ folder is hidden by default in Finder. That's intentional — the engine lives there quietly and you never need to open it.
5
Make sure Adobe Illustrator is installed and open

ColorProof drives the real Illustrator app to read and write artwork. There's no headless mode — Illustrator must be running whenever you use ColorProof. Any recent CC version is fine.

The first time ColorProof runs, macOS will ask if Claude can control Adobe Illustrator. Click OK. If you accidentally declined, go to System Settings → Privacy & Security → Automation and re-enable it.

6
Point Cowork at your artwork folder

In a Cowork session, give Claude access to the folder that contains your logo package (or its parent folder). This is how Claude can read your files and write backups.

In Claude Cowork: Add folder → navigate to your artwork directory → Allow Access

7
Run a smoke test

With Illustrator open and a folder pointed at, tell Claude:

"Proof this package against Orange = CMYK 0/62/100/0."

Claude should open a report in your browser showing the color inventory and pass/fail status. If you see that, setup is complete.

You're done. From here, just describe what you want: "Check these files against our brand colors," "Make the purple Pantone 7683 C everywhere," "Proof this logo package." ColorProof handles the rest.

Troubleshooting

No report appeared after running a job
Two causes: (1) A dialog box is blocking Illustrator — bring Illustrator to the front and dismiss any open alerts. (2) A large package timed out — tell Claude to "run it backgrounded" and it will use a longer-timeout method automatically.
A Pantone color came out as a muddy placeholder
The Pantone book on your machine needs to be found. Make sure LogoPackageExpress is installed in Illustrator (it ships the Pantone library ColorProof reads). If a specific Pantone number is very new, tell Claude the CMYK build or hex value instead and it will use that.
Claude says it can't control Adobe Illustrator
Go to System Settings → Privacy & Security → Automation and make sure the Claude/Cowork entry is allowed to control Adobe Illustrator. Toggle it off and back on if it looks enabled but isn't working.
A raster file (.jpg or .png) didn't update
The file's name needs to match its source .ai or .svg by base name, and include a pixel size token like _1200px or _2160px in the filename. Rename to match that pattern and re-run.
How do I revert a fix if something went wrong?
Every fix takes a whole-folder snapshot to ~/Desktop/ColorProof Backups/ before touching anything. Find the folder with the timestamp from before your run, and copy those files back over the originals. Nothing is ever permanently overwritten.
A spot color readback shows CMYK values instead of its Pantone name
This is normal and expected. When a Pantone spot lives in a CMYK document, Illustrator stores its preview color in CMYK. The separation plate name (what actually prints) is still correct — ColorProof uses the plate name, not the preview color.
