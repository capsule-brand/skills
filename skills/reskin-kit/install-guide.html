Reskin Kit

Restyle a project to match a reference's look and feel, animation included, without touching app logic

v1.0.0
What this does. Reskin Kit restyles your current project to match a reference site, template, or screenshot, capturing both the look (color, type, spacing, components) and the feel (motion, easing, scroll behavior). It writes a DESIGN.md and a MOTION.md, detects the reference's animation stack, then ports the styling and motion into your presentation layer while leaving your routes, auth, data, and business logic untouched. Give it a live URL when you can, since only a running page reveals motion.

This is a one-time setup. After it, you just describe the reskin in plain language.

How you actually use it (just describe the job)

  • Reskin this landing page to match the look and feel of stripe.com.
  • Make this app feel like that reference, including the scroll animation.
  • Capture the design and motion spec from this URL so I can reuse it later.
  • Layout from site A, color from site B, motion from site C.
  • Restyle just the hero section to match this screenshot.

1
Get the skill file

You need reskin.skill. Download it from the catalog and save it somewhere easy to find (Downloads is fine).

Note: If your file picker won't accept a .skill file, rename it to reskin.skill.zip for the upload, then rename it back if needed.
2
Turn on Code Execution in Claude

Reskin writes spec files and edits your project, so it needs Claude's code feature on. In Claude: Settings → Feature Preview → turn on "Code execution & file creation." Once only.

3
Upload the skill to Claude

Go to Settings → Claude Cowork → Upload skill and choose reskin.skill. You'll then see Reskin in your skills list.

Invoke it just by describing the reskin ("make this match the feel of that site"). If you use Claude Code, the same kit ships as the /reskin and /capture-design slash commands.
4
Connect Claude to your Chrome

To match a reference's motion, Reskin inspects the live page in a browser: computed styles plus the running animation stack (GSAP, Framer Motion, Lenis, Lottie, Three.js). So connect Claude in Chrome: install the Claude browser extension and connect the Chrome you'll point it at.

Note: A screenshot captures the look only. To port the feel (easing, timing, scroll behavior), give it a live URL.
5
Point Cowork at your project folder

In a Cowork session, share the project you want to restyle: Add folder → choose your project → Allow Access. That lets Claude read your stack (package.json, config) and edit the presentation layer.

It works on a new git branch and only touches styling, components, and animation. Routes, auth, env, and data calls stay exactly as they were.

6
Try it

With Chrome connected and your project shared, say:

"Reskin this page to match the look and feel of stripe.com."

Claude detects your stack, writes DESIGN.md and MOTION.md, ports the styling and motion onto a branch, wires prefers-reduced-motion, then reports what matched, what needs a manual pass, and the branch to review.

You're set. From here just describe the reference and the scope: a single component, one page, or the whole app.

Troubleshooting

Will it change my app logic or break a route?
No. It works on a new git branch and only touches the presentation layer — styling, components, animation. Routes, auth, env, and data calls (e.g. Supabase) stay untouched. Review the branch diff before you merge.
The motion didn't come across
Motion can only be read from a live URL through Chrome. A screenshot gives look only. Hand it the running page's URL so it can capture easing, durations, triggers, and scroll behavior into MOTION.md.
An effect feels heavy or janky after the reskin
Some effects don't reskin cleanly — bespoke GSAP timelines, Three.js/WebGL, canvas. The skill flags those as "needs a manual pass" before it implements, so check that list and budget time for them.
It used a font I'm not licensed for
It shouldn't. Reskin flags commercial or licensed fonts and suggests a license-clean equivalent instead of silently hotlinking one. Pick the substitute, or drop in your licensed font files.
What about reduced motion and contrast?
Every ported animation gets a prefers-reduced-motion fallback, and it won't push text contrast below WCAG AA to match a look — it flags the conflict instead of shipping it.
Can it capture a reference without changing my project?
Yes. In Claude Code that's the read-only /capture-design command: it writes DESIGN.md, MOTION.md, and assets into a folder and changes nothing, so you can build a reference library to feed a later reskin.
Which animation library will it use?
It detects your stack (React/Vue/Svelte, Vite/Next, Tailwind, and so on) and reuses your existing animation lib, or installs the matching one — Framer Motion for component motion, GSAP for scroll timelines, Lenis for smooth scroll. Any bundle cost is noted in the report.
